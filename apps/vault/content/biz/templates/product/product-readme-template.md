## When to Create README

**Required for:**
- Every product (01-22)
- Major features (if standalone)
- Beta/pilot programs

**Not needed for:**
- Minor bug fixes
- Internal tools (unless complex)

---

## README Maintenance

**Update when:**
- Product launches (create initial README)
- Major feature added (version bump)
- Status changes (PLANNED → ACTIVE → DEPRECATED)
- Pricing changes
- Ownership changes

**Review:** Quarterly (ensure accuracy)
