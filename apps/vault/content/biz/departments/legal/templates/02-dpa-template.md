## 1. DEFINISI / DEFINITIONS

**1.1 Data Pribadi / Personal Data:**
Informasi tentang individu yang dapat diidentifikasi, termasuk nama, email, nomor telepon, data transaksi, dan lokasi bisnis.
Information about identifiable individuals, including names, emails, phone numbers, transaction data, and business locations.

**1.2 Pemrosesan / Processing:**
Setiap operasi pada Data Pribadi, termasuk pengumpulan, penyimpanan, analisis, penggunaan, pengungkapan, atau penghapusan.
Any operation on Personal Data, including collection, storage, analysis, use, disclosure, or deletion.

**1.3 Subjek Data / Data Subject:**
Individu yang Data Pribadinya diproses (misalnya, pelanggan restoran, pemilik bisnis).
Individuals whose Personal Data is processed (e.g., restaurant customers, business owners).

**1.4 Insiden Keamanan / Security Incident:**
Pelanggaran keamanan yang menyebabkan penghancuran, kehilangan, perubahan, pengungkapan, atau akses tidak sah terhadap Data Pribadi.
Security breach causing destruction, loss, alteration, unauthorized disclosure, or access to Personal Data.

## 3. KEWAJIBAN PIHAK KEDUA / OBLIGATIONS OF SECOND PARTY

**3.1 Kepatuhan Hukum / Legal Compliance:**
**PIHAK KEDUA** wajib mematuhi UU PDP No. 27/2022 dan semua regulasi terkait perlindungan data pribadi.
**SECOND PARTY** must comply with PDP Law No. 27/2022 and all related personal data protection regulations.

**3.2 Pemrosesan Sesuai Instruksi / Processing per Instructions:**
**PIHAK KEDUA** hanya akan memproses Data Pribadi sesuai instruksi tertulis dari **PIHAK PERTAMA**, kecuali diwajibkan oleh hukum.
**SECOND PARTY** shall process Personal Data only according to written instructions from **FIRST PARTY**, except as required by law.

**3.3 Kerahasiaan / Confidentiality:**
Semua personel **PIHAK KEDUA** yang mengaksesData Pribadi wajib menandatangani perjanjian kerahasiaan.
All **SECOND PARTY** personnel accessing Personal Data must sign confidentiality agreements.

**3.4 Keamanan Teknis dan Organisasional / Technical and Organizational Security:**
**PIHAK KEDUA** wajib menerapkan langkah-langkah keamanan yang sesuai, termasuk:
**SECOND PARTY** must implement appropriate security measures, including:

- [ ] Enkripsi data saat transit dan istirahat / Encryption of data in transit and at rest
- [ ] Kontrol akses berbasis peran / Role-based access control
- [ ] Audit log dan pemantauan / Audit logs and monitoring
- [ ] Firewall dan perlindungan anti-malware / Firewalls and anti-malware protection
- [ ] Backup data reguler / Regular data backups
- [ ] [Other: _______________]

**3.5 Sub-Prosesor / Sub-Processors:**
**PIHAK KEDUA** TIDAK BOLEH melibatkan sub-prosesor tanpa persetujuan tertulis **PIHAK PERTAMA**.
**SECOND PARTY** SHALL NOT engage sub-processors without written consent from **FIRST PARTY**.

Jika diizinkan, **PIHAK KEDUA** bertanggung jawab penuh atas tindakan sub-prosesor.
If permitted, **SECOND PARTY** remains fully liable for sub-processor actions.

**3.6 Notifikasi Insiden Keamanan / Security Incident Notification:**
**PIHAK KEDUA** wajib memberitahu **PIHAK PERTAMA** dalam waktu **24 jam** setelah mengetahui Insiden Keamanan.
**SECOND PARTY** must notify **FIRST PARTY** within **24 hours** of becoming aware of a Security Incident.

Notifikasi harus mencakup / Notification must include:
- Deskripsi insiden / Description of incident
- Jenis dan jumlah Data Pribadi yang terpengaruh / Type and volume of affected Personal Data
- Tindakan mitigasi yang diambil / Mitigation actions taken
- Rencana pemulihan / Recovery plan

**3.7 Penghapusan Data / Data Deletion:**
Setelah berakhirnya perjanjian, **PIHAK KEDUA** wajib menghapus atau mengembalikan semua Data Pribadi dalam waktu **30 hari**, kecuali diwajibkan menyimpan oleh hukum.
Upon agreement termination, **SECOND PARTY** must delete or return all Personal Data within **30 days**, unless legally required to retain.

**PIHAK KEDUA** harus memberikan sertifikat penghapusan tertulis.
**SECOND PARTY** must provide written certification of deletion.

## LAMPIRAN A: RINCIAN TEKNIS / APPENDIX A: TECHNICAL DETAILS

### MEASURES KEAMANAN / SECURITY MEASURES

**PIHAK KEDUA** berkomitmen untuk menerapkan langkah-langkah berikut:
**SECOND PARTY** commits to implementing the following measures:

**1. Kontrol Akses / Access Controls:**
- Multi-factor authentication untuk semua akun / Multi-factor authentication for all accounts
- Akses hanya diberikan berdasarkan prinsip least privilege / Access granted on least privilege basis
- Review akses setiap 90 hari / Access reviewed every 90 days

**2. Enkripsi / Encryption:**
- Data in transit: TLS 1.2 atau lebih tinggi / TLS 1.2 or higher
- Data at rest: AES-256 encryption
- Database encryption aktif / Database encryption enabled

**3. Pemantauan / Monitoring:**
- Log akses disimpan selama 12 bulan / Access logs retained for 12 months
- Deteksi intrusi aktif / Active intrusion detection
- Alerting untuk aktivitas mencurigakan / Alerting for suspicious activity

**4. Backup dan Recovery / Backup and Recovery:**
- Daily automated backups
- Off-site backup storage (terpisah dari lokasi utama / separate from primary)
- Tested recovery procedures (bulanan / monthly)

**5. Pelatihan / Training:**
- Security awareness training untuk semua personel (tahunan / annual)
- Specific data protection training untuk yang menangani Data Pribadi / Specific data protection training for those handling Personal Data
