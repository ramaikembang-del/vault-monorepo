## When to Document a Process

**Document if:**
- Process repeats >3 times/year
- Multiple people need to do it
- Onboarding new team members
- Regulatory/compliance requirement

**Don't document if:**
- One-off task
- Obvious/intuitive
- Changes constantly (waste of time)

## Best Practices

**1. Keep It Simple**
- Write for beginners (assume zero context)
- Use bullet points, not paragraphs
- Add screenshots for complex UIs

**2. Maintain Regularly**
- Review quarterly
- Update when process changes
- Archive outdated docs

**3. Make Discoverable**
- Store in Notion (central hub)
- Tag appropriately (Finance, Sales, etc.)
- Link from related docs
