## Executive Summary

**Pillar 3 Mission:** Maximize revenue from existing customers by helping them scale their businesses and naturally graduate to higher-value tiers as they grow.

**Strategic Importance:**
- **Higher LTV:** Expansion revenue has 0 CAC (already acquired customers)
- **Predictable Growth:** Easier to forecast than new customer acquisition
- **Product Validation:** Customers paying more = product delivering more value
- **Capital Efficiency:** Expansion ARR (Annual Recurring Revenue) is the best kind of growth

**Expansion Metrics (Targets):**
- **Net Revenue Retention (NRR):** >110% (expansion exceeds churn)
- **Upsell Rate:** 20%+ of customers upgrade within 12 months
- **Cross-Sell Rate:** 30%+ buy additional products/add-ons
- **ARPU Growth:** +15% YoY (average revenue per user increasing)

## Expansion Products & Mechanisms

### E1: Web Dashboard Lite (Freemium Upgrade Path)

**Free Tier Limits (Designed to Drive Upgrades):**
- 30-day data history (Phase 1 Mikro only needs recent data)
- 1 outlet/location
- 1 user login
- 6 basic features only
- Manual CSV upload (no auto-sync)

**Paid Tier ($200-350K/mo) Unlocks:**
- 12-month history (enables trend analysis)
- 2 outlets (expanding businesses)
- 2-3 users (delegating tasks)
- 14 features (full analytics suite)
- WhatsApp alerts (proactive insights)

**Expansion Trigger:** User hits 30-day limit â†’ In-app message: "Upgrade to see last year's data and spot trends"

**Conversion Rate Target:** 5-10% of free users upgrade within 90 days

## Expansion Triggers & In-App Prompts

### Usage-Based Triggers

**Trigger 1: Data Volume Limit**
- Free Lite: 30-day history reached
- **Prompt:** "You've tracked 30 days! Upgrade to see 12 months of trends. Rp 200K/mo"

**Trigger 2: Feature Limit**
- User tries to use locked feature (e.g., WhatsApp alerts)
- **Prompt:** "WhatsApp daily summaries available in Paid tier. Upgrade for Rp 350K/mo"

**Trigger 3: User Limit**
- User tries to invite 4th teammate (Basic allows 3 max)
- **Prompt:** "Need more users? Upgrade to Pro for 10 team seats. Rp 800K/mo"

**Trigger 4: Outlet Limit**
- User opens 2nd outlet, wants to track both
- **Prompt:** "Track all outlets in one dashboard. Upgrade to Pro for Rp 1.2M/mo"

---

### Time-Based Triggers

**Onboarding Upsell (Day 7):**
- User completes onboarding, actively using product
- **Prompt:** "You're a power user! Unlock advanced features with Pro. 30-day free trial?"

**Renewal Upsell (Month 11):**
- User approaching annual renewal
- **Prompt:** "Renew 1 year of Pro and save 20%. Lock in Rp 800K/mo rate."

**Seasonal Upsell (Tax Season):**
- March-April (Indonesian tax filing season)
- **Prompt:** "Get bank-ready P&L reports with Pro Pack. Tax season discount: 50% off first month"

---

### Behavioral Triggers

**Power User Pattern:**
- User logs in daily, uses 90% of features
- **Auto-qualify for upsell:** Sales team reaches out with Pro offer

**Team Expansion:**
- User invites 2nd user (Basic allows 3, they have room to grow)
- **Soft Prompt:** "Planning to add more team? Pro supports 10 users."

**Feature Request:**
- User submits feature request that exists in higher tier
- **Response:** "Good news! That's already in Pro tier. Want a demo?"

## Expansion Sales Playbook

### Self-Serve Upgrades (Default)

**In-App Upgrade Flow:**
1. User clicks "Upgrade" button in app
2. Shows tier comparison table (current vs upgraded)
3. One-click upgrade (no sales call needed)
4. Instant access to new features

**Target:** 80% of upgrades should be self-serve (frictionless)

---

### Sales-Assisted Upgrades (High-Touch)

**When to Involve Sales:**
- Enterprise tier ($800K+ MRR potential)
- Custom dashboard projects (Rp 3-8M)
- 5+ outlet multi-location customers
- API integration requests

**Sales Process:**
1. **Inbound Lead:** Customer requests upgrade or hits limit
2. **Discovery Call:** Understand needs (15-30 min, WhatsApp or Zoom)
3. **Demo:** Show Pro features solving their specific problems
4. **Proposal:** Custom pricing based on outlets, users, features needed
5. **Close:** Contract signed, onboarding scheduled

**Close Rate Target:** 40%+ of sales-assisted leads close

---

### Expansion Pricing Strategy

**Principle:** Make saying "yes" easy (small incremental cost for big value unlock)

**Good Upsell Pricing:**
- Lite Free â†’ Lite Paid (Rp 200K): 0 â†’ 200K (infinite % increase, but low absolute)
- Lite Paid â†’ SaaS Basic (Rp 400K): +Rp 200K (2x, but unlocks team/cloud)
- SaaS Basic â†’ Pro (Rp 1M): +Rp 600K (2.5x, but unlocks multi-outlet)

**Bad Upsell Pricing:**
- SaaS Basic (Rp 400K) â†’ Pro (Rp 3M): 7.5x increase feels like punishment

**Rule:** Each tier should be 2-3x previous tier, not 5-10x

## Net Revenue Retention (NRR) Model

**NRR Formula:**
```
NRR = (Starting MRR + Expansion MRR - Churn MRR) / Starting MRR

Example (100 customers at Rp 400K/mo = Rp 40M MRR):
Starting MRR: Rp 40M
+ Expansion: 20 customers upgrade (+Rp 600K avg) = +Rp 12M
- Churn: 10 customers cancel (-Rp 400K avg) = -Rp 4M
Ending MRR: Rp 48M

```

## Cultural Considerations (Indonesia)

**"Naik Kelas" Messaging:**
- Upgrades = status elevation, not price gouge
- Frame as: "You've graduated to the next level!" (celebrate growth)

**Soft Upsell:**
- Don't use aggressive popups or block features entirely
- Offer upgrade, but allow continued use if they decline (trust-based)

**Bundle Value:**
- Indonesians love bundles/packages ("paket hemat")
- Frame as: "Save Rp 125K/month with bundle!" (not "pay more")

**Annual Prepay:**
- Offer big discount for annual payment (cash flow preference)
- Example: Pay 10 months, get 12 months (17% discount)
