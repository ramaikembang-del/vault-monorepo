# R2c COGS - Variance Investigation SOP

**Owner:** Customer Success
**Last Updated:** December 2025

---

## Purpose
Help customers investigate and resolve COGS variances.

## Variance Analysis

### Acceptable Variance
**Target:** Theoretical vs Actual COGS within Â±5%

### When to Investigate
- Variance > 5% for single product
- Variance > 10% for product category
- Consistent variance trend (3+ weeks)

---

## Investigation Process

### Step 1: Identify Variance
**Dashboard Alerts:**
- Red flag if variance > threshold
- Drill down to product level

### Step 2: Root Cause Analysis
**Common Causes:**
| Cause | Indicator | Fix |
|-------|-----------|-----|
| Over-portioning | Actual > Theoretical | Retrain staff, enforce portions |
| Waste/Spillage | Actual > Theoretical | Reduce waste, better handling |
| Theft/Shrinkage | Actual > Theoretical | Inventory controls, camera review |
| Recipe incorrect | Consistent variance | Update recipe to match actual |
| Data entry error | Sudden spike | Correct inventory logs |

### Step 3: Corrective Action
- Implement fix (retraining, process change)
- Monitor variance next week
- Validate improvement

### Step 4: Document
- Log root cause
- Log action taken
- Track before/after variance

---

## Quality Checks
- [ ] All variances > 10% investigated
- [ ] Root cause identified
- [ ] Corrective action implemented
- [ ] Variance reduced to acceptable level
