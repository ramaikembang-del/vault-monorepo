# R2a P&L Analytics - Configuration SOP

**Owner:** Implementation Team
**Last Updated:** December 2025

---

## Purpose
Configure R2a P&L Analytics module for new customers.

## Configuration Steps

### 1. Chart of Accounts Mapping
- Map customer accounts to P&L categories
- Revenue, COGS, OpEx, Other Income/Expense
- Validate mappings with customer

### 2. Reporting Periods
- Set fiscal year start month
- Configure comparative periods (YoY, MoM)
- Set budget vs actual if available

### 3. Dashboards Setup
- Enable relevant views (location, product, time)
- Configure KPIs (gross margin %, operating margin %)
- Set alert thresholds (margin < target)

### 4. User Training
- Navigate P&L dashboard
- Drill-down analysis
- Export reports

---

## Quality Checks
- [ ] Chart of accounts mapped 100%
- [ ] Numbers reconcile to accounting system
- [ ] Customer trained and can navigate
