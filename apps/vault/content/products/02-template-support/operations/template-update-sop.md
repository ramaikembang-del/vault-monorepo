# R1 Template Support - Template Update SOP

**Owner:** Product Team
**Frequency:** Quarterly + as needed
**Last Updated:** December 2025

---

## Purpose

Manage template version updates and customer communications.

## Scope

**Product:** R1 - Template Support
**Update Types:** Feature updates, bug fixes, performance improvements

---

## Update Process

### 1. Plan Update (2 weeks before)
- Gather feature requests from customers
- Prioritize bug fixes
- Design new features
- Create release plan

### 2. Develop & Test (1 week)
- Update template files
- Test in isolated environment
- Document changes
- Create upgrade guide

### 3. Communicate to Customers (1 week before)
**Email Template:**
```
Subject: A1 Template Update v2.1 Coming Next Week

Hi [Customer],

We're releasing a new template version with:
- [Feature 1]
- [Feature 2]
- [Bug fix]

Release Date: [Date]
Downtime: None (download at your convenience)
Upgrade Guide: [Link]

Questions? Reply to this email.

PWBI Team
```

### 4. Release & Support (Release day + 2 weeks)
- Upload new template to customer portal
- Monitor support tickets for update-related issues
- Provide upgrade assistance if needed

---

## Version Tracking

**Format:** vMajor.Minor.Patch
**Example:** v2.1.3
- Major: Breaking changes
- Minor: New features
- Patch: Bug fixes

**Changelog Maintained:** In template README + customer portal

---

## Quality Checks

- [ ] New version tested before release
- [ ] Release notes documented
- [ ] Customers notified 1 week advance
- [ ] Upgrade guide provided
- [ ] Support team briefed on changes
