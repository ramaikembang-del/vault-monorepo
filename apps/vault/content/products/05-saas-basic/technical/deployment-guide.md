# C1 Full SaaS - Deployment Guide

**Deployment Model:** Cloud SaaS (PWBI-hosted)
**Timeline:** 4-8 weeks (pilot), 3-6 months (full rollout)

---

## Phase 1: Planning (Week 1-2)

- [ ] Kickoff meeting
- [ ] Define scope (locations, users, modules)
- [ ] Identify stakeholders
- [ ] Map data sources (POS, accounting, etc.)
- [ ] Set success criteria

---

## Phase 2: Pilot Location (Week 3-6)

**Choose 1-2 pilot locations:**
- Strategically important
- Manageable size
- Willing to provide feedback

**Activities:**
1. Data integration setup
2. User accounts created
3. Training (4 hours)
4. Go-live
5. 2 weeks of intensive support

**Success Criteria:**
- 90%+ users active
- Data accuracy validated
- Key workflows working
- Positive user feedback

---

## Phase 3: rollout Remaining Locations (Week 7-20)

**Batch approach:**
- Week 7-10: Batch 1 (5 locations)
- Week 11-14: Batch 2 (5 locations)
- Week 15-20: Batch 3 (remaining)

**Per Batch:**
1. Data setup (automated)
2. User training (recorded sessions)
3. Go-live
4. Support

---

## Phase 4: Stabilization (Week 21-24)

- Monitor usage
- Address issues
- Optimize performance
- Gather feedback
- Plan enhancements

---

## Technical Requirements

**Customer Side:**
- Internet connection (min 10 Mbps)
- Modern browsers (Chrome, Safari, Edge)
- Mobile devices (iOS 14+, Android 10+)

**PWBI Side:**
- Cloud infrastructure (AWS)
- Data migration tools
- Integration connectors

---

## Support During Deployment

**Dedicated Team:**
- Implementation Manager
- Technical Lead
- CS Manager

**Support Channels:**
- Slack channel (real-time)
- Weekly check-in calls
- Email support (< 4 hour response)
