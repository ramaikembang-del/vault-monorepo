## E1 Lite Tier-Specific Legal Terms

### **Feature Limitations (ToS Section 5.1)**

| Restriction | Lite Tier | Paid Tiers (Basic+) |
|-------------|-----------|-------------------|
| **Metrics** | 6 metrics | 30-150+ metrics |
| **Locations** | Single outlet | Multi-location |
| **Data Retention** | 90 days | Unlimited |
| **Storage** | 100 MB | Unlimited |
| **Transaction Limit** | 1,000/month | Unlimited |
| **Support** | Community (email) | SLA-backed (<48h) |
| **API Access** | None | Advanced+ tiers only |
| **Uptime SLA** | None (best-effort) | 99.0-99.5% |

### **Upgrade Triggers (ToS Section 5.2)**

Customer **MUST upgrade** if:
- Opens 2nd location  ->  Requires Pro tier
- Exceeds 1,000 transactions/month  ->  Requires Basic tier
- Needs >90 days historical data  ->  Any paid tier
- Requires support SLA  ->  Pro+ tier

### **Termination Rights (ToS Section 5.3)**

Provider may terminate Lite users if:
- Inactivity >180 days (6 months no login)
- Abuse of free tier (fake accounts, scraping)
- Commercial use exceeds "reasonable personal use"

**Notice:** 30 days advance email (except for abuse)

## Fair Use Policy (Lite Tier)

**Acceptable Free Use:**
[x] Personal restaurant analytics (1 location)
[x] Exploratory use (evaluating before upgrading)
[x] Small business (up to 1,000 transactions/month)

**NOT Acceptable (Must Upgrade):**
[x] Multi-location chains using Lite
[x] High-volume restaurants (>1,000 trans/month) avoiding payment
[x] Reselling Lite access to sub-customers
[x] Using multiple Lite accounts to circumvent limits

**Enforcement:** Account suspension + mandatory upgrade OR termination

## Data Retention & Privacy (Lite Tier)

**Data Retention:**
- Lite: 90 days rolling window
- Paid tiers: Unlimited retention

**What Happens to Old Data:**
- Automatically deleted after 90 days (irreversible)
- Customer should export monthly if they need historical data
- Upgrade to paid tier to preserve all historical data

**Data Export:**
- Lite users: Manual CSV export only
- Paid users: Automated export + API access

---

## Upgrade Path (Lite  ->  Paid)

**Self-Service Upgrade:**
1. Dashboard: Click "Upgrade" button
2. Select tier (Basic, Pro, Advanced)
3. Enter payment details
4. Instant activation (no data migration needed - data persists)

**Prorated Billing:**
- First month: Pay only for remaining days
- Example: Upgrade on Jan 15  ->  Pay 50% of monthly fee (15 days remaining)

**Data Preservation:**
- All Lite data transfers to paid account
- Historical data beyond 90 days becomes accessible (if previously deleted, still gone)
