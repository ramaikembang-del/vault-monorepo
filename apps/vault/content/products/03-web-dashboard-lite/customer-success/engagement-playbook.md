# E1: Customer Success - First 7 Days Engagement Playbook
**Purpose:** Maximize activation and early engagement for free users
**Timeline:** Days 0-7 post-signup
**Goal:** 50%+ users open app 3+ times in Week 1

---

## Why First 7 Days Matter

**Critical period for freemium products:**
- Day 0-3: Determines activation (60% should complete onboarding)
- Day 4-7: Determines engagement (50% should become habitual users)
- Day 8+: Engagement patterns set—hard to change after Week 1

**Statistics:**
- Users who engage 3+ times in Week 1 → 80% likely to stay active Month 1
- Users who engage <2 times in Week 1 → 90% churn by Month 1

**Implication:** Win or lose the user in first 7 days.

## Day 0: Signup & Installation

### Goal
User installs app and completes onboarding within 24 hours.

### Automated Touchpoints

**Immediately after signup (email):**
```
Subject: Welcome to Dashboard Lite!

Halo [Name],

Terima kasih sudah join Dashboard Lite!

Langkah berikutnya:
1. Download app: [Link for Android/Windows]
2. Login dengan email ini: [email]
3. Input omzet kemarin (30 detik)
4. Lihat dashboard pertama Anda!

Butuh bantuan? Reply email ini atau WhatsApp: [number]

Salam,
[Your Name]
[Company]
```

**If NOT installed within 6 hours (WhatsApp):**
```
Halo [Name]!

Sudah download Dashboard Lite? Kalau ada masalah install, saya bisa bantu.

Link download: [link]

Atau mau video call 5 menit? Free kok
```

### Success Metric
- **Target:** 70% install within 24h
- **Minimum:** 50% install within 48h

## Day 1: Activation (Onboarding)

### Goal
User completes tutorial and views first dashboard.

### Onboarding Flow (In-App)

**Step 1: Welcome (30 sec)**
```
Screen: "Selamat Datang!"

Text: "Dalam 2 menit, Anda akan lihat:
 Grafik omzet 7 hari
 Top 5 menu terlaris
 Performa staff

Ayo mulai! →"
```

**Step 2: Data Input (60 sec)**
```
Screen: "Input Data Pertama"

Text: "Mulai dengan data kemarin:

Omzet kemarin: Rp ______
Transaksi kemarin: ______

(Atau skip, import dari POS nanti)"
```

**Step 3: First Dashboard View (30 sec)**
```
Screen: Dashboard tampil!

Text: "Ini dashboard Anda!

[Revenue graph shown dengan data dummy/real]

Tap setiap kartu untuk detail.

Tips: Input data tiap pagi biar grafik lengkap!"
```

### WhatsApp Follow-Up (4PM same day)

**If activated:**
```
Selamat! Dashboard Anda sudah aktif

Sudah lihat grafiknya? Besok jangan lupa input omzet hari ini ya!

Ada pertanyaan? Reply aja.
```

**If NOT activated:**
```
Halo [Name], sudah install tapi belum mulai?

Saya bisa video call 10 menit bantuin setup kalau mau.

Reply 'YA' kalau perlu bantuan!
```

### Success Metric
- **Target:** 60% complete onboarding Day 1
- **Minimum:** 40% by end of Day 2

---

## Day 2: Habitual Action (Data Input)

### Goal
User inputs Day 2 data → establishes daily habit.

### Nudge Strategy

**Morning (9 AM WhatsApp):**
```
Selamat pagi [Name]!

Jangan lupa input omzet kemarin biar grafik Anda update

Cuma 30 detik kok! Open app → Input → Done
```

**If data inputted (Evening, 6 PM):**
```
Bagus! Data hari ini sudah masuk.

Lihat grafiknya yuk—omzet naik atau turun vs kemarin?

[Screenshot comparison if possible]
```

**If NOT inputted by 8 PM:**
```
Belum sempat input omzet hari ini?

Gak papa, besok aja. Tapi kalau rutin input, grafiknya jadi lebih berguna!

Mau reminder tiap pagi jam berapa?
```

### Success Metric
- **Target:** 50% users input data Day 2
- **Minimum:** 30% input by Day 3

---

## Day 3: Feature Discovery

### Goal
User explores beyond revenue graph (discovers Top 5 Menu or Staff Performance).

### Feature Spotlight (WhatsApp)

**Message:**
```
Halo [Name]! Sudah tau fitur Top 5 Menu belum?

Cara pakai:
1. Tap "Menu" di dashboard
2. Input 5 menu yang paling laku kemarin
3. Lihat mana yang paling sering dipesan

Berguna buat tau menu favorit customer! Try it?
```

**If user taps Menu feature (in-app message):**
```
Great! Sekarang Anda bisa track menu mana yang paling laris.

Pro tip: Kalau food cost tinggi tapi laku terus, mungkin harganya terlalu murah
```

### Success Metric
- **Target:** 40% users view 2+ metrics by Day 3
- **Minimum:** 25% multi-feature usage

---

## Day 4-5: Value Realization

### Goal
User sees Aha! moment—dashboard provides actual business insight.

### Insight Delivery (WhatsApp)

**If user has 3+ days of data:**
```
Halo [Name]!

Sudah 3 hari track data. Coba lihat:

Grafik omzet 3 hari terakhir—ada pattern gak? (Weekday turun, weekend naik?)

Kalau mau, saya bisa bantuin analisa data Anda (gratis, via call 15 menit).

Interested?
```

**Personalized insight (if available):**
```
[Name], saya lihat omzet Sabtu-Minggu 30% lebih tinggi dari weekday.

Apa Anda udah optimalin marketing weekend? Atau tambah staff weekend?

Dashboard bisa bantu track ini!
```

### Check-In Call (Optional, High-Value Users)

**Criteria for manual outreach:**
- User with >Rp 50M/month revenue (high LTV potential)
- User engaged 5+ times in 5 days (power user signal)
- User asked questions via WhatsApp (engaged, wants more)

**Call script (15min):**
1. "Gimana pengalaman pakai dashboard sejauh ini?" (5min listen)
2. "Ada fitur yang kurang atau bingung?" (3min feedback)
3. "Mau coba fitur Premium gratis 14 hari?" (2 min pitch if appropriate)
4. "Next steps: [Custom recommendation]" (5min action plan)

### Success Metric
- **Target:** User reports 1 actionable insight by Day 5
- **Qualitative:** "Dashboard ini berguna!" sentiment

## Week 1 Engagement Cohort Analysis

### Segmentation

**High Engagement (3+ opens, multi-feature, data input 5+ days):**
- % of cohort: Target 30%
- Action: Upgrade offer (Premium trial Day 8)
- LTV potential: High

**Medium Engagement (2-3 opens, single feature, data input 2-4 days):**
- % of cohort: Target 40%
- Action: Feature discovery nudges (Days 10-14)
- LTV potential: Medium

**Low Engagement (<2 opens, no data input, or 1-time only):**
- % of cohort: Expected 30%
- Action: Reactivation campaign (Week 2) or let churn
- LTV potential: Low

## Optimization Experiments

### Test 1: Onboarding Flow
- **Variation A:** 3-step onboarding (current)
- **Variation B:** 1-step onboarding (skip tutorial, dive straight in)
- **Hypothesis:** Shorter onboarding → higher activation
- **Measure:** Activation rate Day 1

### Test 2: Reminder Timing
- **Variation A:** Morning reminder (9 AM)
- **Variation B:** Evening reminder (6 PM - after closing)
- **Hypothesis:** Evening better (right after they have yesterday's data)
- **Measure:** Data input rate Day 2

### Test 3: Gamification
- **Variation A:** Streak badges ("7 day streak!")
- **Variation B:** No gamification
- **Hypothesis:** Gamification increases habitual usage
- **Measure:** Week 1 engagement rate

## Success Indicators (Week 1 Summary)

### Green Flags (Healthy Cohort)
- >60% activation (complete onboarding)
- >50% Week 1 engagement (3+ opens)
- >30% data input 5+ days
- Positive sentiment ("Dashboard berguna!")

### Yellow Flags (Needs Improvement)
- 40-60% activation
- 30-50% engagement
- <30% consistent data input
- Mixed sentiment

### Red Flags (Failing)
- <40% activation
- <30% engagement
- <20% data input
- Negative sentiment ("Ribet", "Gak berguna")

**Action if Red:** Immediate UX audit, user interviews (5-10 churned users), product iteration
