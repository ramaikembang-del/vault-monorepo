# E1: Objection Handling (Freemium Model)

## Common Objections & Responses

### 1. "Gratis selamanya kan?" (Free forever, right?)

**Context:** User worried about bait-and-switch

**Response:**
"Ya, 100% gratis selamanya untuk 6 metrik dasar:
 Revenue harian
 Top 5 menu
 Basic staff performance
 Customer count
 Average transaction
 Busiest hours

Tidak akan pernah kami cabut. Gratis ya gratis.

Kalau nanti butuh fitur lebih (custom metrics, export PDF, multi-user), baru ada paket berbayar Rp 250K/bulan. Tapi 6 metrik dasar tetap gratis selamanya."

---

### 2. "Kenapa harus bayar?" (Why should I pay?)

**Context:** User doesn't see value in Premium

**Response:**
"Paket gratis sudah cukup kalau Mas/Mba cuma butuh:
- Track omzet harian
- Lihat menu terlaris
- Monitor staff basic

Tapi kalau nanti butuh:
 Export PDF untuk laporan ke owner/investor
 Lihat data 1 tahun penuh (gratis cuma 30 hari)
 Custom metric (misal: profit margin per menu)
 Akses untuk manager (gratis cuma 1 user)

Baru upgrade ke Premium Rp 250K/bulan.

Kebanyakan user pakai gratis 2-3 bulan dulu, baru upgrade kalau udah ngerasa perlu."

**See also:** `pricing-justification.md` for full value breakdown

---

### 3. "POS saya sudah ada report gratis"

**Context:** Comparison to built-in POS reports

**Differentiation:**
"Betul, POS memang ada report. Tapi biasanya:
- Cuma sales data (transaksi, omzet)
- Gak ada analisis (menu mana yang laris? Staff mana yang paling produktif?)
- Gak bisa custom (misal: profit margin, waste tracking)
- Setiap POS beda-beda formatnya

Dashboard Lite kasih:
 Analisis mendalam (bukan cuma angka mentah)
 Visual yang gampang dibaca (grafik, bukan tabel Excel)
 Offline di HP (bisa buka tanpa internet)
 Bisa gabungin data dari beberapa sumber (POS + manual input)

Jadi POS report + Dashboard Lite = lengkap."

---

### 4. "Saya gak ngerti analytics"

**Context:** Intimidation / technical anxiety

**Response:**
"Makanya kami buat yang gampang!

Gak perlu ngerti analytics. Cukup:
1. Input omzet kemarin (30 detik)
2. Lihat grafik di dashboard (sudah otomatis)
3. Baca insight ("Menu Ayam Geprek laris hari ini")

Kalau stuck, kami ada video tutorial 2 menit + WhatsApp support gratis.

Banyak user kami yang gak pernah pakai analytics sebelumnya, tapi dalam 1 minggu udah lancar."

---

### 5. "Nanti saya pikir dulu"

**Context:** Hesitation / not urgent

**Response:**
"Gak masalah! Gak ada pressure.

Kalau Mas/Mba mau, saya bisa:
1. Kirim video demo 2 menit (biar lihat dulu sebelum install)
2. Kasih trial account (bisa coba tanpa install)
3. Jadwalin call 10 menit kalau ada pertanyaan

Atau kalau mau coba langsung, tinggal install—100% gratis, no credit card.

Mana yang lebih nyaman?"

**Follow-up (3 days later):**
"Halo Mas/Mba, sudah sempat pikir tentang dashboard?

Ada pertanyaan yang bisa saya bantu jawab?"

---

### 6. "Datanya aman gak?" (Is my data safe?)

**Context:** Privacy / data security concern

**Response:**
"Aman! Dashboard Lite pake model OFFLINE:
 Data di HP Anda sendiri (not di server kami)
 Gak perlu internet untuk buka dashboard
 Anda yang pegang datanya, bukan kami

Kami cuma nyimpen:
- Email login (buat akun)
- Nama restoran (buat ident

ifikasi)

Data omzet, menu, staff → 100% di perangkat Anda.

Kalau mau lebih aman lagi, bisa pakai fitur 'Local Only' (gak sync ke cloud sama sekali)."

---

### 7. "Terlalu ribet install"

**Context:** Friction / effort concern

**Response:**
"Install cuma 3 langkah (5 menit total):

1. Download dari [link] (seperti WhatsApp)
2. Buka app, masukkan email
3. Input omzet kemarin → Selesai!

Kalau Mas/Mba mau, saya bisa video call 10 menit bantuin install.

Atau saya kirim video tutorial step-by-step (2 menit)."

---

### 8. "Saya tunggu dulu kalau udah sempurna"

**Context:** Waiting for v1.0 / perfectionism

**Response:**
"Saat ini memang masih beta (6 fitur dasar), tapi:
 Sudah bisa pakai untuk track omzet & menu harian
 Gratis selamanya (early adopter benefit)
 Kami update fitur baru tiap 2 minggu
 Beta user dapat prioritas support

Kenapa join sekarang bagus:
- Dapat pengaruh fitur apa yang kami build next (kami dengerin feedback user)
- Dapat premium features gratis 3 bulan (nanti kalau launch resmi, ini berbayar)

Jadi gak rugi coba sekarang—gratis + dapat bonus."

---

### 9. "Harga Premium terlalu mahal"

**Context:** Price objection

**Response:**
"Boleh tahu kenapa terasa mahal? Biar saya bisa jelaskan value-nya.

Untuk context:
- Accounting software (Accurate): Rp 300K-1M/bulan
- Power BI consultant: Rp 30-50M one-time
- Excel manual (staff time): ~10 jam/minggu = Rp 400K/bulan (opportunity cost)

Dashboard Premium Rp 250K/bulan kasih:
 12 custom metrics (vs 6 gratis)
 Export PDF otomatis (hemat 2 jam/bulan)
 1 year data history (buat analisa trend)
 3 user access (owner + manager + admin)

ROI: Kalau hemat 5 jam/bulan (@ Rp 50K/jam) = Rp 250K sudah balik modal.

Mau coba free trial 14 hari dulu? (No credit card)"

**See:** `pricing-justification.md` for detailed ROI breakdown

---

### 10. "Bisa buat beberapa outlet gak?" (Multi-outlet)

**Context:** Scaling concern

**Response:**
"Bisa!

**Gratis:** Cuma untuk 1 outlet
**Premium:** Bisa 1 outlet + aggregated view (lihat total dari semua outlet, tapi gak per-outlet breakdown)
**Pro (Rp 350K/mo):** Full multi-outlet (2-5 outlets, each with separate dashboard + consolidated view)

Kalau Mas/Mba ada 2-3 outlet, saya rekomen langsung pakai Pro.

Atau coba gratis dulu untuk 1 outlet, nanti kalau cocok baru upgrade ke Pro."

---

### 11. "POS saya gak supported"

**Context:** Integration concern

**Response:**
"Gak masalah! Dashboard Lite bisa:

**Option 1: Manual input** (5 menit/hari)
- Input omzet kemarin
- Input top 5 menu sales
- Done

**Option 2: CSV import** (kalau POS export data)
- Export dari POS (format apapun)
- Upload ke dashboard
- Otomatis terbaca

**Option 3: API integration** (kalau POS punya API)
- Kami bisa integrate (Premium/Pro feature)
- Kontak kami untuk setup (1-2 hari)

Mayoritas user pakai manual input—cuma 5 menit/hari, gak ribet kok."

---

### 12. "Nanti aja kalau udah besar"

**Context:** "Too early for analytics"

**Response:**
"Justru sekarang waktu terbaik!

Kalau tunggu sampai besar:
 Data lama udah hilang (gak bisa bandingin trend)
 Kebiasaan buruk udah terbentuk (susah ubah)
 Kesempatan optimasi udah kelewat (misal: menu yang sebenarnya rugi tapi masih dijual bertahun-tahun)

Kalau mulai sekarang:
 Punya data history lengkap (buat keputusan data-driven)
 Bisa optim sejak dini (hemat biaya, naik profit)
 Kebiasaan track metrics jadi culture

Plus, gratis sekarang. Nanti kalau mahal, minimal udah punya data 6-12 bulan."

## Objection Handling Framework (General)

### Step 1: Acknowledge
"Betul, saya ngerti concern Mas/Mba..."

### Step 2: Clarify
"Boleh saya tanya, kenapa merasa [objection]?"

### Step 3: Address
[Provide specific response based on objection type]

### Step 4: Confirm
"Apakah penjelasan ini menjawab concern Mas/Mba?"

### Step 5: Next Step
"Kalau begitu, mau coba [CTA: install / free trial / demo]?"

---

## When to Escalate (vs Self-Serve)

**Self-serve (automated WhatsApp):**
- Objections #1-3, #6, #7 (common, standard response)

**Human touchpoint (sales call):**
- Objection #9-12 (pricing, multi-outlet, integration—higher complexity)
- Any objection mentioned 3+ times by same user (persistent hesitation)

**Escalate to product:**
- User says "Kalau ada fitur X, saya pasti pakai" → Log feature request
